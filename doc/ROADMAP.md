# MergeWong 实现路线图

## P0：可靠同步基础

- [ ] 统一项目名称：Go module、导入路径、二进制、Docker、页面和文档从 `apiwong` 改为 `mergewong`。
- [ ] 明确首版仅支持 MySQL → MySQL，其他连接标为实验性。
- [x] 新增独立全量与 CDC 检查点，不再从目标业务表推导。
- [ ] 任务增加源主键、目标主键、批大小、轮询间隔、写入模式等配置。
- [ ] 使用 `(incremental_key, primary_key)` 复合游标和稳定排序分页。
- [x] 目标端实现批量 upsert，确保失败重试幂等。
- [x] 增加任务级互斥和 CDC context 取消。
- [x] 目标提交成功后推进检查点。
- [x] 校验并引用表名、列名。
- [ ] 为检查点、分页边界、字段映射、重复执行补单元和集成测试。

验收标准：进程在任意批次被终止并重启后，不丢行；重复数据可接受但不会产生重复记录；同一任务不会并发执行。

## P1：全量初始化和可观测性

- [x] 全量任务按主键分页、分批提交，不使用单个长事务。
- [ ] 增加任务运行实例表，区分一次执行与聚合状态。
- [x] 展示表级初始化行数、百分比、当前检查点、吞吐和延迟。
- [x] 支持暂停、恢复和修改 Binlog 位点。
- [x] 支持运行中新增表独立初始化、追数并合并主链路。
- [ ] 密码支持密钥轮换；默认账号和 JWT secret 改为部署时初始化。
- [ ] 增加 Docker Compose 集成测试环境。

## P2：MySQL binlog CDC

- [ ] 定义统一 ChangeEvent 和 Reader/Writer/Transformer 接口。
- [x] 使用 Go binlog 客户端读取 row-based binlog，持久化 file/position。
- [x] 支持 insert/update/delete 和位点恢复。
- [x] 实现全量初始化与 Binlog 位点衔接。
- [x] 在管理端检查 Binlog 配置、ROW/FULL 行镜像和账号权限。
- [ ] 用故障注入验证进程崩溃、网络闪断和目标库超时。

验收标准：在故障恢复和事件重放下目标表最终与源表一致，物理删除能够传播。

## P3：按需求扩展

- PostgreSQL WAL / SQL Server CDC Reader；
- PostgreSQL / SQL Server 方言化批量 Writer；
- 自定义转换表达式或受控 Go/WASM 插件；
- 多实例任务租约、分片和集中指标；
- 对接 Prometheus、告警和审计日志；
- 数据校验、抽样比对与自动修复。

## 暂不建议做

- 一开始就支持所有数据库两两同步；
- 自研消息队列或分布式事务；
- 为“实时”追求毫秒延迟，却没有先完成幂等和检查点；
- 仅因为成熟项目使用 Java 就重写现有 Go 服务。
