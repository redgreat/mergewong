# 同步任务向导与多表同步实施计划

## 目标

把当前单表任务表单演进为分步任务向导，支持批量选择表、逐表修改目标库表名称，并在创建任务前完成可解释的链路预检查。仍以表和数据同步为边界，不复制 DTS 的实例迁移、事件迁移等复杂能力。

## 第一阶段：整理现有单表任务配置

- 分为“基础配置、同步对象、执行与预警”三个步骤。
- 页面不常驻显示说明文字；执行计划和预警策略通过问号弹层解释。
- 保留单表源表、目标表改名和字段改名。
- 调度、预警和连接用途继续使用现有后端能力。

验收：现有任务可以无损编辑；新增任务不再需要在一个长表单中来回查找字段。

## 第二阶段：多表对象选择与映射

当前进度：已完成表清单读取、搜索多选、逐表目标表改名、独立元数据保存和待预检查状态；字段级逐表编辑与可靠多表执行仍需后续阶段完成。

- 新增 `sync_task_tables` 元数据表，一条任务关联多条表映射。
- 左侧读取源连接的表清单，支持搜索和多选；右侧展示已选对象。
- 每个对象维护源库、源表、目标库、目标表、源主键、目标主键、增量字段和字段映射。
- 支持批量设置目标库前缀、表名前后缀，但最终结果逐表可见、可修改。
- 执行器以表映射为最小隔离单元，单表失败不能让其他表的检查点含糊推进。

验收：一条任务可选择多张表并分别改名；任务详情能明确展示每张表的配置和运行结果。

## 第三阶段：创建前预检查

当前进度：已完成 MySQL → MySQL 的连接、权限、表、单列主键、增量字段、目标字段和基础类型兼容检查；检查通过后任务自动启用。

新增只读预检查接口，至少验证：

1. 源端、目标端网络和账号可连接。
2. 连接用途与权限匹配：源端可读，目标端可建表或写表。
3. 源表存在，目标库存在；目标表冲突策略明确。
4. 表名、列名、主键和增量字段通过白名单及方言引用校验。
5. 增量任务具有稳定的 `(增量字段, 主键)` 排序条件。
6. 字段映射无重复目标列，关键字段没有被遗漏。
7. 源目标字段类型兼容；不兼容项必须阻止创建或明确确认转换规则。
8. 全量任务估算行数和表大小，提示长事务与磁盘风险。

预检查结果分为“阻断、警告、通过”，阻断项未解决时不能启用任务。

## 第四阶段：可靠执行与可恢复性

当前进度：已完成任务互斥、每表独立检查点、500 行分批、复合游标读取和 MySQL 主键 upsert；物理删除仍明确留给后续 CDC Reader。

- 每张表使用独立检查点，不再读取目标表 `MAX()`。
- 使用 `(增量字段, 主键)` 复合游标稳定分页。
- 目标端按主键批量 upsert，实现 at-least-once 下的幂等重跑。
- 增加任务级互斥和表级运行状态；每批目标提交成功后才推进对应检查点。
- 明确软删除字段方案；物理删除留到 CDC Reader 实现。

验收：批次任意中断后可恢复，同游标值不丢行，重复执行不产生重复记录。

## 不在本计划内

- 双向同步。
- 数据库事件、存储过程、函数和账号权限迁移。
- 把轮询增量描述为 CDC 实时同步。
